## LESSON PLAN TEMPLATE

When I ask Claude to plan a single lesson, follow this format:

**Lesson title:** [Clear, outcome-focused — e.g., "How to Write a Hook That Stops the Scroll"]
**Lesson goal:** [One sentence — what the student can do after this lesson]
**Duration:** [Target length in minutes]

**Structure:**
1. Hook (0:00-1:00) — [Why this lesson matters. What goes wrong if they skip it.]
2. Concept (1:00-4:00) — [The core idea. One concept only. No tangents.]
3. Example (4:00-7:00) — [Show it working in real life. Use my own examples, not hypotheticals.]
4. Tutorial (7:00-11:00) — [Step-by-step. Screen recording or demonstration.]
5. Common mistakes (11:00-12:00) — [The 2-3 things students get wrong. Show the fix.]
6. Action step (12:00-12:30) — [Exactly what the student does right now, before the next lesson.]

**Resources to include:**
- [Template / worksheet / checklist — if applicable]
- [Recommended tool / link — if applicable]

## SALES PAGE STRUCTURE

When I ask Claude to write a sales page, use this order:

1. **Headline** — [Pain-aware or desire-aware. Specific outcome. No fluff.]
2. **Sub-headline** — [Who this is for. One sentence.]
3. **The problem** — [Agitate what they're struggling with. Specific, not generic.]
4. **The solution** — [How my course solves it. What makes it different.]
5. **What's inside** — [Module breakdown. Each module gets one benefit sentence, not a description.]
6. **Who this is for / not for** — [Honest. Both sections.]
7. **Results / proof** — [Student wins, my credentials, numbers if I have them]
8. **FAQ** — [Answer the real objections: time, price, level, guarantee]
9. **CTA** — [Clear. One option. No confusion about next step.]

## LAUNCH CHECKLIST

Before I publish any course:

- [ ] Every lesson has exactly one outcome
- [ ] Every module ends with an action step
- [ ] Sales page has been tested on someone outside my niche
- [ ] There are no lessons over 15 minutes without a break or quiz
- [ ] Student support process is documented (how they ask questions, turnaround time)
- [ ] Welcome email is written and automated
- [ ] Course platform tested on mobile
- [ ] Refund/cancellation policy is written and visible
